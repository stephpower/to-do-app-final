Deployment Guide · To-Do App

Troubleshooting Your Deployment

Follow these steps in order. Not every step will apply to you — read each one carefully before moving on.

1
Find Your Vercel Domain
Check
▼

Before making any code changes, find the domain Vercel assigned to your frontend. You'll need this exact value for the next two steps.

Find your Vercel domain
  • Go to vercel.com/dashboard and open your to-do-app-frontend project
  • Click Overview in the left sidebar
  • Copy the short domain under Domains — not the long Deployment URL above it
Keep this domain handy — you'll paste it into backend/index.js in the next step.
2
Fix the CORS URL in backend/index.js
Common fix
▼

In your backend index.js, find the origin value inside your CORS configuration and update it to match the Vercel domain you copied in Step 1.

Wrong vs. Correct
✕  Incorrect
origin: 'https://to-do-app-frontend-indol.vercel.app/',
✓  Correct
origin: 'https://to-do-app-frontend-indol.vercel.app',
⚠Remove the trailing / at the end of the URL. The presence of a slash breaks CORS and prevents frontend-backend communication.
Use the exact Vercel domain you copied in Step 1 — no trailing slash.
3
Update API_URL in frontend/js/api.js and frontend/js/auth.js
Required
▼

Both api.js and auth.js inside frontend/js/ contain a hardcoded API_URL that must point to your live Render backend URL. If this is wrong, your frontend can't communicate with your backend at all.

Find your Render backend URL
  • Go to dashboard.render.com and open your backend web service
  • Copy the URL shown at the top — it looks like https://to-do-app-backend-xxxx.onrender.com

4
Check Environment Variables on Render
Required
▼

Your Render environment variables must exactly match what is in your local .env file — including the variable names.

Required variables
⚠
Do not include quotes around the values in Render. No quotes around the MONGODB_URI or SECRET_KEY.
⚠
The variable name must match exactly — MONGODB_URI not MONGO_URI. A single typo here will break the database connection.
Do not edit your .env file in VS Code. The .env file must stay inside the backend folder only.
Get your MongoDB URI from Compass
  • Open MongoDB Compass on your desktop
  • Click the ··· button next to your connection
  • Select "Copy connection string"
  • Paste it into the Render environment variable (no quotes)
Optional: click Connect in MongoDB Compass on . If it connects successfully, your string is valid and your database is active.
5
Push Changes to GitHub
Required
▼

After making the above changes, push your code. You must be in the correct root folder first.

Confirm you're in the right folder

Your terminal should be inside to-do-app/ — the folder that contains both frontend/ and backend/. Check by running:

pwd ls
You should see backend/ and frontend/ listed. If not, navigate to the correct folder first.
Push commands — run one at a time
git add . git commit -m "updated deployment fixes" git push origin main
⚠After pushing, Vercel and Render will automatically redeploy. Wait a few minutes before testing.
Test your app using the Vercel domain only — not localhost.
6
Still broken? Create a New GitHub Repo
Last resort
▼

If nothing above fixed the issue, your original repository was likely pushed from the wrong folder. Delete everything and start completely fresh.

1. Delete your old GitHub repository
  • Go to your old repository on GitHub
  • Click Settings at the top of the repo page
  • Scroll to the very bottom — the Danger Zone section
  • Click "Delete this repository" and follow the confirmation prompt
2. Delete your Vercel and Render projects
  • In Vercel — open your frontend project → Settings → scroll to bottom → Delete Project
  • In Render — open your backend service → Settings → scroll to bottom → Delete Service
Deleting the Render service means your backend URL will change.
You'll need to update API_URL in api.js and auth.js again once the new Render service is live (Step 3).
Deleting and recreating your Vercel project will give you a new Vercel domain — in this case, you'll also need to update the CORS origin in backend/index.js to match it (Step 2).
3. Create a new GitHub repo and push
  • Create a new repository on GitHub
  • In VS Code, open the terminal and navigate to your to-do-app root folder
  • Copy the HTTPS URL from the blue Quick Setup box on GitHub (shown below) — it will look like https://github.com/your-username/your-new-repo.git
  • Run the commands below, replacing the URL with your new repo URL
git init git add . git commit -m "first commit" git remote add origin https://github.com/your-username/your-repo.git git branch -M main git push -u origin main
⚠The git remote add origin line is what connects your local folder to your new GitHub repo. Make sure the URL here is your new repo URL — not the old one. If you're unsure, run git remote -v to check which repo you're currently connected to.
💡If you see an error saying remote origin already exists, run git remote remove origin first, then re-run the git remote add origin command with your new URL.
4. Reconnect Vercel and Render
  • Create a new project in Vercel and connect it to your new GitHub repo frontend folder
  • Create a new web service in Render and connect it to your new GitHub repo backend folder
  • Re-add your environment variables in Render
  • Then repeat Steps 1–5, making sure to update your new Render URL in api.js and auth.js
